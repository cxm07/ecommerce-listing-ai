# 协作规则

1. 修改前阅读相关 `docs/`，并遵守其中的公共契约。
2. 不得单方面改动公共数据结构、API 字段或任务状态机；此类改动必须使用独立 Pull Request。
3. 路由、前端、模型和 Agent 不得绕过 `WorkflowService` 推进状态。
4. 前端不得暴露高权限密钥；模型或 Agent 不得猜测缺失商品事实；原始上传文件不得覆盖。
5. 外部数据先转为统一商品结构；ERP 和平台只能经 Connector；Agent 只能调用白名单 Skill。
6. 所有新功能必须有测试，且不得修改当前任务无关模块。
7. 每次任务结束报告修改文件、测试结果和遗留问题。
