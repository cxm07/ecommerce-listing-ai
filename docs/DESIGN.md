# 前端设计基线

本文件定义电商商品上新 AI 工作台的前端呈现规范。它只约束视觉、交互和可访问性；不定义或修改 API、公共数据结构、权限模型和任务状态机。

## 设计原则

- **审核优先**：界面帮助用户理解当前任务、待处理问题和下一步，不以装饰掩盖业务状态。
- **克制可信**：使用深墨导航、浅雾内容画布和单一蓝靛强调色；不使用营销页式大渐变、炫光或高饱和多色装饰。
- **状态可读**：成功、提醒、失败、当前步骤和禁用操作均使用语义色、文字和形状共同表达，不能只依赖颜色。
- **密度适中**：列表和审核数据保持可扫描性；卡片用于分组，不为每个字段单独包裹卡片。

## Token

`frontend/src/styles.css` 是 Token 唯一来源，所有页面应使用语义变量而非新增散落的十六进制色值。

| 类别 | Token | 用途 |
| --- | --- | --- |
| 内容画布 | `--canvas`、`--surface`、`--surface-subtle` | 页面、面板和次级区域 |
| 文字 | `--ink`、`--ink-secondary`、`--ink-muted` | 标题、正文、辅助信息 |
| 边界 | `--line`、`--line-strong` | 面板、表格和控件边界 |
| 主操作 | `--primary`、`--primary-hover`、`--primary-soft` | 当前选择和主要动作 |
| 状态 | `--success`、`--warning`、`--danger`、`--info` 及对应 `-soft` | 审核、问题、解析和反馈 |
| 节奏 | `--space-1` 至 `--space-12` | 4px 基数的间距体系 |
| 形状 | `--radius-sm`、`--radius-md`、`--radius-lg` | 8 / 12 / 16px 圆角 |

正文与交互文本必须保持至少 4.5:1 的对比度。阴影仅使用 `--shadow-low` 表达面板层次，不能与宽重阴影叠加制造浮夸卡片效果。

## 共用组件规范

- `AppShell` 负责主导航、页面标题、操作区、跳到主要内容链接和页面容器。侧栏底部只显示“演示环境”，不得展示 Mock、API 或仓储实现信息。
- 所有主页面使用同一个 `PageContainer`（由 `AppShell` 提供）并保持最大宽度 1480px。
- 主按钮使用 `primary-button`，次要按钮使用 `soft-button`，文字操作使用 `link-button`；禁止为相同语义发明新的按钮外观。
- 面板使用 `panel`，默认 12px 圆角、细边框和低阴影。不要在同一个页面堆叠多层圆角卡片。
- `WorkspaceStep` 永远渲染六个等宽节点。连线只在圆形节点之间；已完成为勾选、当前步骤为蓝靛实心节点、未到达步骤为可见描边圆、失败步骤为红色语义状态。

## 交互与可访问性

- 所有键盘可达控件必须有可见焦点环；触控目标最小高度为 38px，移动端优先保留 40px 以上高度。
- 动画只用于悬停和状态反馈，时长 160ms 左右；`prefers-reduced-motion` 下必须关闭非必要动画。
- 桌面端侧栏固定在 248px；1080px 以下收缩为单列工作区；820px 以下将导航变为页面顶部的紧凑导航，表格和审核区改为单列。
- 加载、空和失败状态必须说明用户当前能做什么，不能以技术实现名称代替业务说明。

## 范围边界

前端页面只能通过既有 Repository 和 Workspace 聚合读取模型获取数据。页面不得直接访问数据库、读取 Mock JSON、拼接跨接口业务事实，或自行推进任务状态。涉及公共字段、API 或状态机的修改必须单独走契约 PR。
