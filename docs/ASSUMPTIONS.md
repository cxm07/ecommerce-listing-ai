# 已知事实与假设

## 已知事实

- 当前仓库用于业务验证型 MVP。
- 本轮的技术方向是 React/TypeScript/Vite、FastAPI/Pydantic/openpyxl 和 Supabase。

## MVP 假设

- 商品数据由一种固定的模拟 Excel 模板提供。
- 商品字段、审核步骤和知识规则只服务演示，不代表企业永久规则。
- 后续输出首先是 Excel。

## 未知问题

- 未做真实企业调研，尚不清楚企业是否使用 ERP、CRM、平台后台、PDF、Word 或图片资料。
- 不清楚真实字段、角色、审批链、最终输出以及 Agent 需求。

## 待验证问题

向老师、运营人员或企业确认：样本模板、必填字段、违规声明、审批角色、数据主系统、导出目标和数据保留要求。未经确认不得将上述假设写死为业务规则。
